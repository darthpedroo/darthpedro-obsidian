Como distinguir entre el hijo y el padre

Si queremos ejecutar exec solo en el hijo, tenemos que DISTINGUIR ENTRE ELLOS. Fork -> Devuelve un número. En el padre este numero es el PID. En el hijo, vale 0. Porque es el hijo.

Que pasa cuando hay un Fork

Se copia toda la pcb del proceso padre, con todos los valores (menos el PID). -> Si hago esa copia, tengo dos procesos que corren el mismo código.

Para diferenciarlos, el valor de retorno cambia según el padre y el hijo. Padre => PID. HIjo => 0.

El hijo resume la ejecución en el mismo punto que el padre, cuando termina de hacer el fork. Como tal, no sabe que es hijo. Lo sabemos nosotros con el PID => 0.

Fork Bomb

int main(){
	while(1) fork;

Vamos a desglosarla:

:(){ :|:& };:
  • :() — define una función llamada : (sí, eso es legal en Bash).
  • { :|:& } — dentro de la función, se llama a sí misma dos veces usando un pipe (|) y ejecutando en segundo plano (&):
    • :|: — llama a sí misma y pasa la salida a otra instancia de sí misma.
    • & — ejecuta el proceso en segundo plano, permitiendo que siga lanzando más instancias.
  • ; — termina la definición de la función.
  • : — llama a la función por primera vez, lo que inicia la reacción en cadena.

Terminación de un proceso

  • El mismo proceso termina normalmente -> este mismo llama a exit
  • El proceso es terminado por otro proceso -> kill()
  • El proceso es terminado por el sistema operativo (falta de recursos, error. etc)

Señales

  • La relación entre padre e hijo permite la intercomunicación entre estos ya que conocen el PID del otro
  • Si se conoce el PID del otro , se pueden MANDAR SEÑALES
  • El padre puede esperar a que el hijo termine
  • kill() se usa para enviar señales a otros procesos y puede ser utilizada para terminar un proceso.

Filicidio

  • Común que un proceso padre mate al hijo al terminar.
  • Esta es una forma de evitar que los procesos hijos queden huerfanos.

Procesos zombies

Proceso huerfano -> Se convierte en un proceso zombie. Si padre hace wait y lo matan -> Cuando el hijo muere , el hijo le avisa al padre que muere (SIEMPRE LO HACE.) El hijo muere pero no se borra de la PCB, porque el PADRE es el que se encarga de eso.
